第一阶段：补齐高频工具
目标是快速提高收藏价值。

PDF 工具组
加入 PDF 合并、拆分、压缩、PDF 转图片、图片转 PDF 增强版。
这是最值得优先做的一组，和“本地处理、隐私安全”的定位非常匹配。

开发者常用工具
新增时间戳转换、Cron 表达式解析、YAML/TOML 格式化、JSON Schema 校验、颜色格式转换、UUID v7/ULID/NanoID 生成。

图片批量处理增强
图片压缩、图片格式转换、图片 Base64、OCR 支持批量文件、队列进度、失败重试、批量下载。

第二阶段：提升工作台体验
目标是让工具不只是“一个个页面”，而是能连续完成任务。

批处理系统
做统一的文件队列组件，支持拖拽上传、进度展示、取消任务、结果列表、批量下载 zip。

工具间跳转
在结果区提供“继续处理”入口，比如：
OCR 结果 → 文本去重 / 文本替换
JSON 格式化 → 代码高亮 / JSON Schema 校验
图片压缩 → 图片格式转换 / 图片转 PDF

历史记录增强
目前已有最近使用，可以进一步支持“最近处理记录”，但只保存元信息，不保存用户敏感内容。

第三阶段：信任感与可用性
目标是强化“纯前端安全工具箱”的品牌感。

隐私提示统一化
在涉及文件、图片、视频、OCR、加密的工具页增加统一提示：本地处理、不上传服务器。
网络类工具单独说明哪些请求会访问外部服务。

PWA 离线支持
支持安装到桌面、离线打开、缓存核心静态资源和常用 WASM 文件。

WASM 加载体验
给 FFmpeg、OCR、ONNX 这类重资源工具加预加载提示、加载进度、失败降级说明。

第四阶段：质量与工程化
目标是让项目更适合长期维护。

抽公共组件
优先抽：文件上传区、结果面板、复制按钮、下载按钮、进度条、工具页标题区、隐私提示条。

加工具模板
提供 tool-template.ts 或脚手架说明，统一新工具的结构、错误处理、toast、清理逻辑。

测试与校验
加基础单元测试覆盖纯函数工具，比如编码、文本转换、哈希参数处理。
再加少量 Playwright 冒烟测试：首页、搜索、路由、几个核心工具可打开。

推荐里程碑
M1：PDF 合并/拆分/压缩 + 时间戳转换 + Cron 解析
M2：统一批处理组件 + 图片批量压缩/转换
M3：PWA 离线 + 隐私提示系统
M4：工具模板 + 基础测试 + 工具间继续处理入口
